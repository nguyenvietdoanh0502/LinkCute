# Rule: Documentation First

## Rule: Always Read Documentation
Before implementing any feature or fixing any bug, you MUST read and understand the relevant documentation in the `document/` folder. This ensures alignment with business requirements, architectural decisions, and project conventions.

## Project Context
The summary of the project context is maintained in `AGENTS.MD` at the project root. This file must be consulted at the start of every session to maintain continuity and understanding of the project's state and goals.
